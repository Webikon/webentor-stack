# External Integrations

**Analysis Date:** 2026-04-11

## APIs & External Services

**Image CDN:**
- Cloudinary - Image resizing, optimization, and delivery via fetch API
  - SDK/Client: Custom `CloudinaryClient` PHP class at `packages/webentor-core/app/CloudinaryClient.php`
  - Auth: `CLOUDINARY_CLOUD_NAME` constant (set via env)
  - Endpoint pattern: `https://res.cloudinary.com/{cloud_name}/image/fetch/{transformations}/{source_url}`
  - Helper functions in `packages/webentor-core/app/images.php`:
    - `get_resized_cloud_picture()` / `get_resized_cloud_picture_by_url()` - `<picture>` element with responsive sources
    - `get_resized_cloud_image()` / `get_resized_cloud_image_by_url()` - `<img>` element with Cloudinary URL
    - `get_resized_cloud_image_url()` / `get_resized_cloud_image_url_by_url()` - URL only
  - Supports crop modes, retina (2x), responsive breakpoints, gravity options
  - Falls back to original URL when `CLOUDINARY_CLOUD_NAME` is not defined
  - Optional `WP_IMAGES_REWRITE_HOME` for localhost URL rewriting

**Error Tracking:**
- Sentry - Error monitoring for both PHP and JavaScript
  - SDK/Client: `wpackagist-plugin/wp-sentry-integration` ^8.2.0 (WordPress plugin via Composer)
  - Auth: `SENTRY_DSN_PHP` (PHP errors), `SENTRY_DSN_JS` (browser errors)
  - Config: `packages/webentor-starter/config/application.php` (lines 159-164)
  - Enabled conditionally when env vars are set

**Search:**
- Typesense - Full-text search engine (optional)
  - SDK/Client: Docker container via `typesense/typesense:29.0` image
  - Auth: `WTC_TS_API_KEY` env var (defaults to `"local"`)
  - Port: `WTC_TS_NODE_PORT` env var
  - Setup: `packages/webentor-setup/typesense-docker.sh` launches Docker Compose
  - Docker Compose generated by `webentor-setup init --with-typesense true`
  - Compose file: `scripts/docker-compose.typesense.yml` (project-level, generated)

**Analytics:**
- Google Tag Manager - Tag management
  - Plugin: `wpackagist-plugin/duracelltomi-google-tag-manager` ^1.20 (`packages/webentor-starter/composer.json`)

## Data Storage

**Databases:**
- MySQL/MariaDB - Primary WordPress database
  - Connection: `DB_NAME`, `DB_USER`, `DB_PASSWORD`, `DB_HOST` env vars, or `DATABASE_URL` DSN
  - SSL support: `DB_SSL` env var enables `MYSQLI_CLIENT_SSL`
  - Client: WordPress `$wpdb` (used directly in `packages/webentor-core/app/blocks-migration.php`)
  - Config: `packages/webentor-starter/config/application.php` (lines 87-106)

**Caching:**
- Redis - WordPress object cache
  - Plugin: `wpackagist-plugin/redis-cache` ^2.0 (`packages/webentor-starter/composer.json`)
  - Connection: `REDIS_HOST`, `REDIS_PORT` env vars
  - Disable: `WP_REDIS_DISABLED` env var
  - Config: `packages/webentor-starter/config/application.php` (lines 151-153)
  - Disabled by default in development: `packages/webentor-starter/config/environments/development.php`

**File Storage:**
- WordPress media library (local filesystem at `web/app/uploads/`)
- Cloudinary fetch API for CDN delivery (does not replace uploads, fetches originals and transforms)

## Content Management

**Custom Fields:**
- Advanced Custom Fields Pro (ACF) - Custom field management
  - Plugin: `wpengine/advanced-custom-fields-pro` ^6.3 (`packages/webentor-starter/composer.json`)
  - Config: `packages/webentor-core/app/acf.php`
  - CPT/Taxonomy registration via ACF disabled (code-first approach)
  - Admin UI hidden (all field groups defined in code)

**Forms:**
- Gravity Forms - Form builder
  - Plugin: `gravityforms/gravityforms` (custom Composer package source)
  - Auth: `PLUGIN_GF_KEY` (license key for download)
  - Anti-spam: `wpackagist-plugin/gravity-forms-zero-spam` ^1.4

**Page Caching:**
- WP Rocket - Full-page cache and performance optimization
  - Plugin: `wp-media/wp-rocket` ^v3.15.10 (`packages/webentor-starter/composer.json`)
  - Auth: `WP_ROCKET_KEY`, `WP_ROCKET_EMAIL` env vars
  - Config: `packages/webentor-starter/config/application.php` (lines 170-175)

**Image Optimization:**
- ShortPixel - Image compression
  - Plugin: `wpackagist-plugin/shortpixel-image-optimiser` ^6.0
- Better Image Sizes - On-the-fly image resizing
  - Plugin: `wpackagist-plugin/better-image-sizes` ^3.4
  - Used by helper functions in `packages/webentor-core/app/images.php` (e.g., `bis_get_attachment_image_src()`)

**SEO:**
- The SEO Framework (autodescription) - SEO meta management
  - Plugin: `wpackagist-plugin/autodescription` ^5.0

**Email:**
- WP Mail SMTP - Outbound email transport configuration
  - Plugin: `wpackagist-plugin/wp-mail-smtp` ^4.0

**Database Migration:**
- WP Migrate DB Pro - Database synchronization between environments
  - Plugin: `deliciousbrains-plugin/wp-migrate-db-pro` ^2.3
  - Source: `https://composer.deliciousbrains.com` Composer repository

## Authentication & Identity

**Auth Provider:**
- WordPress built-in authentication
  - Login URL hidden via `wpackagist-plugin/wps-hide-login` ^1.8
  - User role management via `wpackagist-plugin/user-role-editor` ^4.64

**WordPress Multisite:**
- Supported via `DOMAIN_CURRENT_SITE` env var
  - Config: `packages/webentor-starter/config/application.php` (lines 136-146)
  - URL fixer: `roots/multisite-url-fixer` ^1.1

**Internationalization:**
- Multisite Language Switcher (MSLS) - Multisite-based i18n (optional)
  - Integration: `packages/webentor-core/app/i18n.php`
  - Helper functions: `get_msls_current_lang()`, `get_msls_languages()`
- Loco Translate - Translation management
  - Plugin: `wpackagist-plugin/loco-translate` ^2.6
- WordPress i18n via `@wordpress/i18n` on JS side

## Monitoring & Observability

**Error Tracking:**
- Sentry (see APIs section above)

**Debugging:**
- Tracy (PHP) - Development debugging
  - Package: `tracy/tracy` ^2.10 (dev dependency in `packages/webentor-starter/composer.json`)
- WordPress debug mode: `WP_DEBUG`, `WP_DEBUG_DISPLAY`, `WP_DEBUG_LOG`
- Webentor debug panel: controlled via `WEBENTOR_WP_DEBUG` JS variable (injected in `packages/webentor-core/app/setup-core.php`)

**Logs:**

- PHP `display_errors` off in production, on in development

## CI/CD & Deployment

**Hosting:**
- GitHub (source code, monorepo)
- GitHub Pages (documentation site)
- npm registry (JS packages: `@webikon/webentor-core`, `@webikon/webentor-configs`)
- Packagist (PHP packages: `webikon/webentor-core`, `webikon/webentor-setup`)

**CI Pipeline:**
- GitHub Actions
- Workflows:
  - `.github/workflows/ci.yml` - Lint, build, test per package (path-filtered)
  - `.github/workflows/release.yml` - Changesets version + npm publish on main
  - `.github/workflows/docs-deploy.yml` - VitePress build + GitHub Pages deploy
  - `.github/workflows/split-webentor-core.yml` - Git subtree split to mirror repo on `core-v*` tags
  - `.github/workflows/split-webentor-setup.yml` - Git subtree split on `setup-v*` tags
  - `.github/workflows/split-webentor-starter.yml` - Git subtree split on `starter-v*` tags
  - `.github/workflows/demo-bump.yml` - Demo site version bumping

**Git Hooks:**
- Husky pre-commit (`.husky/pre-commit`):
  - Builds `webentor-core` when source files are staged
  - Auto-stages rebuilt `public/` assets
  - Skipped in CI (`$CI` env var check)

**Package Distribution:**
- npm packages published via Changesets action
- Composer packages distributed via Packagist (mirror repos created by subtree split workflows)
- Packagist auto-pinged after split via API: `https://packagist.org/api/update-package`

## Environment Configuration

**Required env vars (starter projects):**
- `DB_NAME`, `DB_USER`, `DB_PASSWORD` (or `DATABASE_URL`)
- `WP_ENV` (defaults to `development`)
- WordPress salts: `AUTH_KEY`, `SECURE_AUTH_KEY`, `LOGGED_IN_KEY`, `NONCE_KEY`, `AUTH_SALT`, `SECURE_AUTH_SALT`, `LOGGED_IN_SALT`, `NONCE_SALT`

**Optional env vars:**
- `CLOUDINARY_CLOUD_NAME` - Enable Cloudinary image CDN
- `SENTRY_DSN_PHP`, `SENTRY_DSN_JS` - Enable Sentry
- `REDIS_HOST`, `REDIS_PORT` - Enable Redis
- `WP_ROCKET_KEY`, `WP_ROCKET_EMAIL` - WP Rocket license
- `DOMAIN_CURRENT_SITE` - Enable multisite
- `DB_SSL` - MySQL SSL mode

**Secrets location:**
- `.env` file at project root (gitignored)
- Optional 1Password integration: `scripts/.env.setup` stores vault/item IDs
- CI secrets: GitHub Actions secrets (`NPM_TOKEN`, `GITHUB_TOKEN`, `PACKAGIST_USERNAME`, `PACKAGIST_API_TOKEN`, `CORE_MIRROR_TOKEN`, etc.)

## Webhooks & Callbacks

**Incoming:**
- WordPress AJAX endpoints for block migration tool:
  - `wp_ajax_webentor_migration_scan` - Scan posts for v1 attributes
  - `wp_ajax_webentor_migration_run` - Migrate v1 to v2 attributes
  - `wp_ajax_webentor_migration_cleanup` - Remove legacy v1 attribute keys
  - All in `packages/webentor-core/app/blocks-migration.php`
- WordPress standard `admin-ajax.php` URL exposed to frontend via `window.wpAjaxUrl` (`packages/webentor-core/app/setup-core.php`)

**Outgoing:**
- Packagist API ping after subtree split (`.github/workflows/split-webentor-core.yml`)
